# 仓库入口设计原则

这个文档用于约束前端后续设计时，`挂载` 与 `仓库` 两个入口的职责边界。

## 1. `挂载` 按钮关注“仓库如何挂载到存储中”

- 这里放和挂载路径、内部/外部存储、外部设备绑定相关的信息。
- 这里放和“卸载仓库”相关的动作，因为它表达的是把这个仓库记录从当前挂载关系里移除。
- 这里尽量不要放与仓库内容组织、模板、Overlay、预览策略相关的设置。

## 2. `仓库` 按钮关注“仓库本身”

- 这里放仓库名字、绑定模板、Overlay、自定义行为、预览配置等内容。
- 这里放扫描与失效项清理这类“作用于仓库内容”的操作。
- 这里尽量不要重复展示挂载路径和外部设备绑定等挂载层信息，除非只是为了上下文提示。

## 3. 设计目标

- 使用者看到 `挂载` 时，应联想到“这个仓库如何接到存储上”。
- 使用者看到 `仓库` 时，应联想到“这个仓库自身的规则、行为和内容管理”。
- 后续如果继续扩展按钮弹窗或区块，优先沿着这条边界增加，不要把两类职责重新混在一起。
